Important FAQ

1. What is the Document Object Model (DOM)?

DOM stands for Document Object Model. It's a programming interface that represents an HTML document as a tree structure of objects. This tree structure allows JavaScript to access, modify, and create dynamic web content.

2. How do you select an element from the DOM?

In JavaScript, you can select elements from the DOM using various methods:

1. getElementById: This method selects an element based on its unique ID attribute.
2. getElementByClassName: This method selects all elements with a specific class name.
3. getElementByTagName: This method selects all elements with a specific tag name.
4. querySelector: This method selects the first element that matches a CSS selector.
5. querySelectorAll: This method selects all elements that match a CSS selector.

3. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in JavaScript where you attach an event listener to a parent element, allowing it to handle events that bubble up from its child elements. This is more efficient than attaching event listeners directly to each child element, especially when dealing with many elements.

It is useful because,

Performance: It's faster because you create fewer event listeners.
Dynamic Elements: It works well with elements added to the page later.
Simplified Code: It makes your code easier to write and maintain.

4. How do you manipulate an element's attributes and styles using the DOM?

The Document Object Model (DOM) provides a programmatic interface to interact with HTML elements. One of the fundamental operations is manipulating their attributes and styles.

Attributes are key-value pairs associated with HTML elements. They provide additional information or metadata about the element.

Accessing and Modifying Attributes:

1. getAttribute(attributeName): Retrieves the value of a specified attribute.
2. setAttribute(attributeName, value): Sets the value of a specified attribute.
